# 🤖 具身智能柔性关节控制平台 (EJCP)
## 工业级实施标准与真实基线文档 V1.0 (最终核准版)

**发布日期：** 2026年3月30日
**项目状态：** 阶段一、二（工程平台验证、底层固件重写、物理通信打通）已圆满完结。
**架构定义**：采用“分布式解耦架构”。下位机（小脑）负责底层高频 FOC 控制、安全限幅与物理柔顺；上位机（大脑）负责轨迹下发与宏观逻辑。

### 第一部分：硬件系统清单（BOM）与物理接线基线

#### 1. 核心硬件选型
*   **下位机（小脑）**：HW3511 2804 一体化智能无刷模组（集成 STM32F103 主控、DRV8311 驱动、MT6701 高速磁编码器）。自带电流采样，支持真实阻抗控制。
*   **上位机（大脑）**：ESP32-S3-DevKitC-1-N16R8（双核，算力溢出，负责后续复杂 AI 算法）。
*   **供电系统**：12V DC 直流电源适配器（配合 DC 5.5x2.1 转接端子）。

#### 2. 🚫 绝密物理接线规范（绝对真理）
所有通信与烧录**仅使用电机板的 `PORT2` 接口（SH1.0 6P）**，彻底废弃 I2C，采用极其稳定的 UART 协议。
以成品彩色排线为例，**线序与连接规范严格如下**：
*   **Pin 1 (红线 = 3.3V)** ➡️ **【极其致命：严禁接线，必须悬空绝缘！】**（防止双电源打架烧板）
*   **Pin 2 (蓝线 = GND)** ➡️ **接 ESP32 的 `GND` 引脚**（必须共地，消灭乱码的核心）
*   **Pin 3 (绿线 = 电机 TX)** ➡️ **接 ESP32 的 `8` 号引脚 (RX, 听)**
*   **Pin 4 (黄线 = 电机 RX)** ➡️ **接 ESP32 的 `9` 号引脚 (TX, 说)**

#### 🛠️ 修改 1：破解芯片 ID 验证 (解决 `libusb` 与 `UNEXPECTED idcode` 报错)
*   **文件**: `platformio.ini` (约第 43 行)
*   **修改**: 将芯片识别码强制改为 STM32 原厂码。
    ```ini
    upload_flags = -c set CPUTAPID 0x1ba01477
    ```

#### 🛠️ 修改 2：降频通信，消灭乱码
*   **文件**: `src/config.h` (全局搜索 `SERIAL_BAUDRATE`)
*   **修改**: 强行降频至工业级极稳速率。
    ```cpp
    #define SERIAL_BAUDRATE 9600
    ```

#### 🛠️ 修改 3：封杀自动电压检测炸机隐患
*   **文件**: `src/main.cpp` (全局搜索 `readVoltage`)
*   **修改**: 注释该行，强制使用安全的默认电压。
    ```cpp

    ```

#### 🛠️ 修改 4：【核心护城河】将安全与柔顺参数物理刻死！
*   **文件**: `src/main.cpp`，在 `setup()` 函数末尾的 `for` 循环内。
*   **修改**: 替换为以下代码，彻底剥夺上位机配置负担，实现真正的黑盒化交付！

### 第四部分：上位机（大脑）终极调度固件

小脑底层固化后，大脑的通信负担降至最低。大脑只负责下发轨迹，代码**极其极简**。
在 `ESP32_Brain` 工程的 `main.cpp` 中烧录以下 V1.0 发布版代码：

```cpp
#include <Arduino.h>

#define MOTOR_RX_PIN 8  
#define MOTOR_TX_PIN 9  

HardwareSerial MotorSerial(1); 

void setup() {
  Serial.begin(115200); 
  MotorSerial.begin(9600, SERIAL_8N1, MOTOR_RX_PIN, MOTOR_TX_PIN); 
  
  Serial.println("\n=====================================");
  Serial.println("🚀 具身大脑已启动！准备注入【精准到位】参数！");
  
  // ⏳ 黄金 8 秒：绝对不打扰电机的底层磁场对齐与开机自检
  delay(8000); 
  while(MotorSerial.available()) MotorSerial.read(); // 清理开机杂音

  Serial.println("\n✅ 8秒结束！开始恢复动力基因！");
  
  // 💉 1. 恢复力气：最高电压放宽到 5V（克服物理静摩擦力）
  MotorSerial.print("MLU5.0\n"); delay(200);
  
  // 💉 2. 恢复记忆：速度环积分(I)恢复到 5.0（消除稳态误差，死磕到 90度）
  MotorSerial.print("MVI5.0\n"); delay(200);

  // 💉 3. 增强弹簧刚度：设为 4.0（兼顾柔顺感与目标牵引力）
  MotorSerial.print("MAP4.0\n"); delay(200);

  // 💉 4. 保持优雅：最高速度限制在 2.0（防止起步阶跃暴冲和剧烈抖动）
  MotorSerial.print("MVL2.0\n"); delay(200);

  Serial.println("🎉 参数注入完毕！这次绝对指哪打哪！\n");
}

unsigned long lastMoveTime = 0;
bool toggle = false;

void loop() {
  // 🎧 原样打印小脑的日志反馈
  while (MotorSerial.available()) {
    char c = MotorSerial.read();
    Serial.write(c); 
  }
  
  // ⏱️ 周期任务：每 4 秒交替下发目标坐标
  if (millis() - lastMoveTime > 4000) {
    lastMoveTime = millis();
    Serial.println("\n-------------------------------------");
    if (toggle) {
      Serial.println("👉 大脑下令: 稳稳去 90度 (M1.57)");
      MotorSerial.print("M1.57\n"); // 纯净的换行符 \n 结尾
    } else {
      Serial.println("👈 大脑下令: 稳稳回 0度 (M0)");
      MotorSerial.print("M0\n");    
    }
    toggle = !toggle;
  }
}
```

### 第五部分：避坑与排雷沉淀库（Troubleshooting）

这是我们用血泪踩平的天坑，后来者必须熟读：

1. **`Warn: \r detected` 报错导致电机罢工**：
   * **原因**：SimpleFOC 底层协议极其严格，禁止接收 Windows 标准的 `\r\n` 回车符，一旦收到 `\r` 就会抛弃整条指令。
   * **解决**：大脑下发指令强制只用 `\n`。
2. **通信满屏乱码、幽灵乱跳**：
   * **原因**：90% 是假地线陷阱！工厂排线颜色随机，未做到两板“共地 (GND)”；或者是波特率时序错位（如 115200 去听 9600）。
   * **解决**：顺着电机板 `PORT2` 插座物理数针脚（第 2 根必为 GND）。确保两端 `SERIAL_BAUDRATE` 绝对一致。
3. **起步严重“哆嗦”、走到 45 度卡死（稳态误差）**：
   * **原因**：由于 `P=1.0`（太软）且没加阻尼，电机受到静摩擦力影响无法到达目标；若加入极高 `Velocity I`（速度积分），又会导致起步瞬间怒气值爆表，产生剧烈阶跃暴冲。
   * **解决**：通过 `voltage_limit = 2.0` (限流/限压) 和 `velocity_limit = 5.0` (优雅限速)，直接在物理底层“去势”，将机械猛兽驯化为平滑的柔性关节。

### 🚀 终极点火 SOP（运行标准流程）

1. **断电接线**：按规范连接蓝、绿、黄三根线至 ESP32。
2. **大脑供电**：ESP32 接通 USB，打开电脑串口监视器。
3. **动力接通**：**最后一步，插上电机的 12V 动力电源！**
4. **现象验收**：等待 8 秒自检，电机伴随“滴”声自动锁死。用手捏转子，感受海绵般的柔顺弹力。随后，电机严格按照 ESP32 下发的频率，极其丝滑、无声、安全地进行 90 度折返跑。
