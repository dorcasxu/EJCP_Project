# 📄 文档一：《具身智能柔性关节 - 唯一真实基线文档 V1.0》

**发布日期：** 2026年3月30日
**项目状态：** 阶段二（工程平台与物理通信）已完结，准备进入阶段三（碰撞反射）。

### 1. 核心硬件与物理接线基线（绝对真理）
*   **下位机（小脑）**：HW3511 2804 一体化无刷电机模组（STM32 + DRV8311）。
*   **上位机（大脑）**：ESP32-S3-DevKitC-1-N16R8。
*   **唯一通信接口**：电机板 **`PORT2`**（SH1.0 6P 插座）。
*   **黄金线序（以成品彩色排线为准）**：
    *   Pin 1 (红线) = 3.3V ➡️ **[严禁接线，悬空绝缘]**
    *   Pin 2 (蓝线) = GND ➡️ **接 ESP32 `GND` (必须共地！)**
    *   Pin 3 (绿线) = 电机 TX ➡️ **接 ESP32 `8` 号引脚 (RX)**
    *   Pin 4 (黄线) = 电机 RX ➡️ **接 ESP32 `9` 号引脚 (TX)**
    *   动力电源：12V DC 直接接入电机粗红黑线。

### 2. 固件与通信参数基线
*   **电机端固件版本**：`HowayFOC-main`，编译环境 `[env:DRV_MODEL_HW3511_SSI]`。
*   **大脑端固件版本**：`ESP32_Brain` (基于 Arduino 框架)。
*   **全局波特率（极其关键）**：电机与 ESP32 之间的 UART 通信速率**永久锁定为 `9600`**（彻底解决面包板飞线导致的高频丢包和乱码问题）。
*   **指令语法规范**：所有通过串口下发的指令，**必须以 `\n` 结尾**，绝对禁止包含 `\r` 或空格等多余字符（会导致 `Warn: \r detected` 或指令失效）。

### 3. 电机标准启动时序（SOP）
1.  **0 秒**：接通 12V 动力电源。
2.  **0 ~ 8 秒**：小脑进入底层“电流校准与磁场对齐”真空期，期间对任何指令“耳聋”。
3.  **第 8 秒**：大脑 (ESP32) 确认延时结束，开始下发“开机密码”。
4.  **下发顺序**：
    *   `@3\n` (切入角度模式)
    *   `MAP4.0\n` (注入肌肉刚度)
    *   `MVI5.0\n` (注入速度积分，消除稳态误差)
    *   `MVL2.0\n` (限制最高转速，保证动作优雅)
    *   `MLU5.0\n` (放开电压限制，保证力矩充足)
    *   `ME1\n` (强制通电使能，电机瞬间硬化锁死)

### 4. 官方指令字典（已破译）
| 指令集 | 说明 | 作用 |
| :--- | :--- | :--- |
| **`@3`** | 模式切换 | 切入角度（位置）控制模式 |
| **`ME1` / `ME0`** | 使能开关 | `1`=通电发力锁死，`0`=断电瘫软卸力 |
| **`MAPx.x`** | Angle P (刚度) | 决定肌肉有多硬（1.0极软，10.0极硬） |
| **`MADx.x`** | Angle D (阻尼) | 决定肌肉有多黏稠，用于消除到达目标时的震荡 |
| **`MVIx.x`** | Velocity I (积分) | 消除静摩擦导致的稳态误差（不到位现象） |
| **`MVLx.x`** | Velocity Limit | 限制电机最高转速（弧度/秒），防止起步阶跃暴震 |
| **`MLUx.x`** | Voltage Limit | 限制电机最大输出电压（V），直接决定电机最大力气 |
| **`Mx.xx`** | 目标坐标 | 转动到指定的弧度（如 `M1.57` 为转到 90度） |

### 6. 已知问题列表（Harmless 级 / 无害缺陷）
*   **中文字符乱码**：电机板底层固件使用了 GBK 编码打印调试中文（如“机器模式已初始化”），在 ESP32 的 UTF-8 串口监视器下会显示为菱形乱码（如 `M1.5700 `）。
    *   **处理策略**：**完全无视！** 这反而证明物理通信极度畅通。它绝不影响任何底层数字指令（如 `M1.57`）的执行。未来在大脑代码中，我们只需提取数字，过滤掉汉字即可。
*   **`\r` 字符过敏警告**：SimpleFOC 底层解析器对 Windows 标准的 `\r\n` 回车换行符极度敏感，收到 `\r` 会触发 `Warn: \r detected` 并拒不执行。
    *   **处理策略**：已在基线规范中彻底封杀，大脑发送任何指令**仅允许使用 `\n`**。
